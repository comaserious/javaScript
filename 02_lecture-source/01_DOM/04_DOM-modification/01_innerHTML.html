01. innerHTML

        Element.prototype.innerHTML 프로퍼티는 getter,setter 모두 존재하는 프로퍼티이다
        요소노드의 html 마크업을 취득하거나 변경한다
        textContent 프로퍼티는 html 마크업을 무시하고 테스트만 반환하지만,
        innerHTML 프로퍼티는 HTML 마크업이 포함된 문자열을 그대로 반환한다
태그 엘리먼트값을 읽거나, 변경할때 innerHTML속성을 사용한다
    innerHTML 프러퍼티를 사용하면 DOM 조작이 간단하고 직관적이라는 장점이 있다
    사용자한테 입력받은 데이터를 그대로 innerHTML 에 할당하는 것은 보안에 굉장히 취약하다
    악성코드를 html 마크업내에 포함한다면 파싱(번역) 작업에서 그대로 실행될수가 있다

    innerHTML 이라는 프로퍼티에 html 마크업 문자열을 할당할 경우 요소노드의 
    모든 자식을 제거하고 새롭게 할당한 
    html 마크업 문자열을 파싱하여
    변경한다는 단점이 있다

    새로운 요소를 삽입할때, 삽입될 위치를 지정할 수 없다(끽해야 뒤에다 붙이는거 뿐이다)
